# Лабораторная работа №4: Обработка потоковых данных с Apache Kafka

## Описание
Реализована система потоковой обработки событий пользователей (просмотры, добавления в корзину, покупки) с использованием Apache Kafka.  
Producer генерирует синтетические события, consumer собирает метрики, строит дашборд, детектирует аномалии и использует простую ML-модель для предсказания покупки.

## Результаты анализа (на 300+ событиях)
- **Обработано событий:** 300  
- **Уникальных пользователей:** 92  
- **Конверсия view → purchase:** 13.7%  
- **Общая выручка:** $7 320  
- **Топ-продукт по выручке:** laptop ($4 000)  
- **Брошенные корзины:** 40 пользователей  
- **Основное оповещение:** "Много брошенных корзин"

## Технические детали
- **Kafka:** версия 3.9.2, режим standalone  
- **Python:** 3.x, библиотеки: kafka-python, pandas, matplotlib, seaborn, numpy  
- **ОС:** Windows 10/11 (запуск через CMD)  
- **Проблема:** права доступа к `/tmp/kafka-logs` → исправлено изменением `log.dirs` на `C:/kafka/data/kafka-logs` и `dataDir` на `C:/kafka/data/zookeeper`

## Файлы в репозитории
- `producer_demo.py` – генератор событий  
- `consumer_demo.py` – обработчик, метрики, дашборд, ML  
- `user_sessions.csv` – данные о сессиях  
- `metrics_snapshot.json` – метрики в реальном времени  
- `kafka_analytics_dashboard.png` – дашборд (6 графиков)  
- `report_*.txt` – пример текстового отчёта  

## Запуск (кратко)
1. Запустить ZooKeeper и Kafka (см. инструкцию).  
2. Создать топик `user-actions`.  
3. Выполнить `python producer_demo.py` и `python consumer_demo.py`.
